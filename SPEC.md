# SPEC.md

## Background
대표의 생존 상태를 주기적으로 확인하기 위한 알림/판정 시스템이 필요하다. 이 시스템은 대표의 직접 반응(알림 해제)과 워치 상태(수면, 충전)를 함께 고려해 불필요한 알림을 줄이고 실제 반응 여부를 기록해야 한다.

## Problem
- 전화 미응답이 잦아 대표 상태를 수시로 확인하기 어렵다.
- 단순 주기 알림만 사용하면 수면 중/충전 중에도 불필요한 알림이 발생한다.
- 생존 판정 기준과 이벤트 로그가 없어 사후 확인이 어렵다.

## Goals
- 1시간 간격으로 알림을 울려 대표 반응을 유도한다.
- 알림 해제 시점(사용자 상호작용)을 `생존 확인`으로 저장한다.
- 대표가 직접 `1시간/2시간/3시간` 일시정지를 설정할 수 있게 한다.
- 수면 또는 충전 상태일 때는 알림을 자동 스킵한다.
- 워치 측 판정 이벤트를 백엔드(Go)로 전송해 조회 가능하게 한다.

## Non-goals
- 의료 진단 수준의 정확한 생체 데이터 분석
- 긴급 구조 자동 호출(119/보호자 자동 연락)
- iOS/애플워치 지원
- 다중 사용자/조직 관리 기능

## Constraints
- watchOS 권한 정책(알림, HealthKit, 백그라운드 태스크) 준수 필요
- 수면 데이터는 기기/OS 버전에 따라 가용성과 지연이 다를 수 있음
- 네트워크 불안정 환경에서도 로컬 알림은 지속 동작해야 함
- 초기 버전은 1시간 주기를 기본으로 하며 정확 알람 권한에 의존하지 않는 방향 우선

## Functional Requirements
1. 시스템은 기본적으로 1시간 간격 알림을 스케줄링한다.
2. 사용자가 알림을 해제하면 해당 시점을 `alive` 이벤트로 기록한다.
3. 사용자는 워치 화면에서 `1시간`, `2시간`, `3시간` 일시정지를 설정할 수 있다.
4. 일시정지 기간 동안 알림은 전송되지 않는다.
5. 워치가 수면 상태면 알림을 전송하지 않는다.
6. 워치가 충전 중이면 알림을 전송하지 않는다.
7. 알림 미전송 시 사유(`sleeping`, `charging`, `snoozed`)를 기록한다.

## Success Criteria
- 24시간 동안 알림/스킵/해제 이벤트가 누락 없이 기록된다.
- 사용자가 알림을 해제하면 5초 이내 `alive` 이벤트가 저장된다.
- 수면 또는 충전 상태에서 잘못 울리는 알림 비율이 5% 미만이다.
